# AI Engineering Journey 🚀

My path from absolute beginner to production-ready **AI Engineer**, built in public.

This repo is my single source of truth and my portfolio-in-progress. It follows a mentor-directed, phase-by-phase curriculum designed around how the AI Engineer role is *actually* hired and practiced in 2026 — production-first, portfolio-driven, and reliability-focused.

> 📍 **The full curriculum lives in [`AI_Engineer_Roadmap.md`](AI_Engineer_Roadmap.md).**

---

## What I'm becoming

An **AI Engineer**: a builder of production software where AI models are the core component. The goal isn't demos in notebooks — it's shipping reliable, observable, cheap, and safe AI features and products. The defining skill is *building reliable systems out of unreliable components.*

## The 2026 stack I'm learning

- **Core:** Python, Git, FastAPI, Docker, pytest, async
- **LLM app layer:** LLM APIs, Pydantic, structured outputs, tool calling
- **RAG:** embeddings, pgvector / Qdrant / Weaviate / Pinecone, re-ranking, RAGAS
- **Agents:** LangGraph, CrewAI, Pydantic AI, MCP; ReAct, Plan-Execute, multi-agent, HITL
- **Eval & Ops:** LLM-as-judge, golden sets, LangSmith / Langfuse; CI/CD, cost & latency monitoring
- **Fine-tuning:** Hugging Face `transformers` + `peft` + `trl`; LoRA / QLoRA
- **Cloud:** AWS / GCP / Azure
